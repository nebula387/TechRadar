# TechRadar AI — CLAUDE.md

## Project Vision

Build a fully automated, multi-platform content system called **TechRadar AI**.
The system discovers trending tech content, rigorously filters it for quality,
and distributes polished posts across multiple channels simultaneously.

**Core principle: publish LESS but BETTER. Quality over quantity.**

---

## Output Channels

| Channel | Language | Format |
|---|---|---|
| Telegram Channel | Russian 🇷🇺 | Short post + image card |
| Website (static blog) | English 🇬🇧 | Full article page |
| Reddit | English 🇬🇧 | Post to r/MachineLearning, r/programming, r/artificial |
| X (Twitter) | English 🇬🇧 | Thread (3–5 tweets) |
| LinkedIn | English 🇬🇧 | Professional post |

---

## Tech Stack

- **Language:** Python 3.12+
- **HTTP:** `httpx` (async) + `requests` (sync fallback)
- **Data validation:** `pydantic v2`
- **Database:** `SQLite` via `SQLModel`
- **Scheduler:** `APScheduler`
- **LLM (classify/filter):** Groq API — **free tier only**
- **LLM (content generation):** OpenRouter API — **free models only**
- **Image cards:** `Pillow`
- **Website:** static HTML/JSON generated by Python (no framework needed for MVP)
- **Publishing:** Telegram Bot API, Reddit PRAW, Tweepy (X), LinkedIn API
- **Config:** `.env` via `python-dotenv`

### Free LLM Models to Use

**Groq (fast, for filtering):**
- `llama-3.3-70b-versatile` (free tier)
- `gemma2-9b-it` (free tier)

**OpenRouter (for content generation, use `:free` suffix):**
- `mistralai/mistral-7b-instruct:free`
- `google/gemma-2-9b-it:free`
- `meta-llama/llama-3.1-8b-instruct:free`
- `qwen/qwen-2-7b-instruct:free`

**Rule:** Never use paid models. If free quota is exhausted — skip publishing, log warning, retry next cycle.

## Core Data Models (`app/models.py`)

```python
from pydantic import BaseModel
from datetime import datetime
from enum import Enum

class Source(str, Enum):
    GITHUB = "github"
    HUGGINGFACE = "huggingface"
    PRODUCTHUNT = "producthunt"
    HACKERNEWS = "hackernews"
    ARXIV = "arxiv"

class Category(str, Enum):
    DEVELOPER_TOOL = "Developer Tool"
    AI_MODEL = "AI Model"
    RESEARCH_PAPER = "Research Paper"
    OPEN_SOURCE = "Open Source"
    AI_AGENT = "AI Agent"
    MCP_SERVER = "MCP Server"
    DATASET = "Dataset"

class RawItem(BaseModel):
    source: Source
    title: str
    url: str
    description: str
    stars: int | None = None
    language: str | None = None
    tags: list[str] = []
    author: str | None = None
    raw_data: dict
    collected_at: datetime

class ScoredItem(RawItem):
    score: int                  # 0–100 (only items >= 85 get published)
    category: Category
    audience: str               # e.g. "ML Engineers", "Python Developers"
    is_interesting: bool
    novelty: int                # 0–10: is this genuinely new/unique?
    practical_value: int        # 0–10: can someone use this today?
    wow_factor: int             # 0–10: would an expert find this impressive?
    reason: str                 # why it's worth publishing
    rejection_reason: str | None = None
    emoji: str
    accent_color: str           # hex color for card

class GeneratedContent(BaseModel):
    item: ScoredItem
    # Per-channel content
    telegram_text_ru: str       # Russian, Markdown
    twitter_thread_en: list[str]  # list of tweets (280 chars each)
    linkedin_post_en: str       # professional tone, English
    reddit_title_en: str        # catchy Reddit title
    reddit_body_en: str         # Reddit post body
    website_title_en: str       # SEO-optimized title
    website_body_en: str        # Full article (300–500 words)
    website_slug: str           # URL slug
    tags: list[str]
    image_path: str | None = None

class PublishedRecord(BaseModel):
    item_url: str
    source: Source
    score: int
    published_at: datetime
    channels: list[str]         # which channels it was published to
    telegram_message_id: int | None = None
    reddit_post_id: str | None = None
    tweet_id: str | None = None
```

## Content Quality Filter (CRITICAL)

This is the most important part of the system. **Most items must be rejected.**

### Stage 1 — Hard Pre-filter (`filter/quality_gate.py`)

Reject immediately (no LLM call) if ANY of these are true:

```python
HARD_REJECT_RULES = [
    lambda item: item.stars is not None and item.stars < 500,  # GitHub: too small
    lambda item: len(item.description) < 30,                    # no description
    lambda item: item.title.lower().count("tutorial") > 0,      # skip basic tutorials
    lambda item: item.title.lower().count("awesome-") > 0,      # skip awesome lists
    lambda item: is_already_published(item.url),                 # dedup check
]
```

Accept to next stage only if:
- GitHub: stars > 500 OR today's growth > 300 stars
- HuggingFace: trending rank < 20 OR downloads > 10k
- HackerNews: score > 200
- ProductHunt: upvotes > 100
- ArXiv: from top venue OR highly cited preprint

### Stage 2 — LLM Judge (`filter/llm_judge.py`)

Use **Groq** (fast, free). Prompt:

You are an expert tech curator with extremely high standards.
Your job is to decide if the following item is worth publishing to an audience
of senior developers and ML engineers.
Be VERY strict. Reject anything that is:

Generic or "another X tool"
A tutorial or beginner content
Already well-known (existed > 6 months without major update)
Lacks concrete practical value
Pure marketing / startup announcement without substance

Only approve items that are:
✅ Genuinely novel or a meaningful breakthrough
✅ Solves a real pain point in an elegant way
✅ Would make an expert say "wow, I didn't know this existed"
✅ Has significant community traction (stars, upvotes, citations)
✅ Relevant to AI, ML, or developer productivity
Item:
Title: {title}
Description: {description}
Stars/Score: {metric}
Source: {source}
Respond with ONLY valid JSON:
{
"approve": true/false,
"score": 0-100,
"novelty": 0-10,
"practical_value": 0-10,
"wow_factor": 0-10,
"category": "...",
"audience": "...",
"reason": "one sentence why approved/rejected",
"emoji": "single emoji",
"accent_color": "#hex"
}
Minimum score to approve: 85. If score < 85, set approve: false.

**Expected rejection rate: 70–85% of all collected items.**

## LLM Content Generation (`llm/generate.py`)

Only called for approved items (score >= 85). Use **OpenRouter free models**.

### Telegram (Russian)

You are the editor of a popular Russian Telegram channel about AI and developer tools.
Write a post in Russian that is:

Engaging and conversational (not corporate)
3–5 sentences max
Starts with the emoji: {emoji}
Bold title in first line
Explains WHAT it is and WHY it matters RIGHT NOW
Ends with a link

Item: {item_data}

### Twitter Thread (English)

Write a Twitter thread (3–5 tweets) about this tech item.
Tweet 1: Hook — make people stop scrolling. Under 280 chars.
Tweet 2-4: Key insight, how it works, why it matters. Each under 280 chars.
Last tweet: Call to action + link.
Be punchy. No corporate speak. Talk like a senior engineer sharing a find.
Item: {item_data}

### LinkedIn Post (English)

Write a LinkedIn post (150–250 words) about this item.
Tone: professional but not boring. Share a genuine insight.
Structure: hook → context → what makes it interesting → practical takeaway → link.
No hashtag spam. Max 3 relevant hashtags at the end.
Item: {item_data}

### Reddit Post (English)

Write a Reddit post for r/{subreddit}.
Title: under 300 chars, specific and informative (not clickbait).
Body: 100–200 words. Explain clearly what it is, what problem it solves,
and why it's relevant now. Reddit readers are technical — be precise.
No hype language.
Item: {item_data}
Subreddit: {subreddit}

### Website Article (English)

Write a short tech article (300–400 words) about this item for a developer blog.
Structure:

Headline (SEO-optimized, under 70 chars)
Lead paragraph (what it is, why it matters)
Key features / what makes it unique (2–3 paragraphs)
Who should care and practical use cases
Closing thought

Tone: expert, clear, no fluff. Target reader: senior developer or ML engineer.
Item: {item_data}

Each post page includes:
- Full English article
- Image card
- Source link
- Tags
- Score badge
- Links to all social posts (Telegram, Reddit, Twitter, LinkedIn)

The `public/` folder can be deployed to GitHub Pages, Netlify, or Cloudflare Pages for free.

## Code Standards

- All functions: full type hints
- All API keys: from `.env` only, never hardcoded
- Async-first: use `asyncio` + `httpx` for all network calls
- Each publisher: independently togglable via config flag
- Log every LLM call: model name, input length, latency, cost estimate
- Exponential backoff on all API calls (max 3 retries)
- If any publisher fails: log error, continue to next publisher (don't crash)
- `tests/` for: quality_gate logic, LLM JSON parsing, dedup logic

---

## Important Notes for Claude Code

- **Never hardcode API keys**
- **Use only free LLM models** — check the free model list above
- **The quality filter is the heart of the system** — implement it carefully
- **Each publisher must be independent** — failure in one must not affect others
- The website is static HTML — no Flask/FastAPI needed for MVP
- Start with only Telegram + Website enabled, add Reddit/Twitter/LinkedIn after MVP works
- The system should publish **0–3 posts per day** — silence is better than noise
